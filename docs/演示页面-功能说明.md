# 演示页面 - 功能说明

本文汇总项目中的各类演示与调试页面：智能客服「模拟对方连发」、SSR 对比、test-ws、Protobuf、stress、Server Actions 等，便于快速定位与理解用途。

---

## 一、智能客服：模拟对方连发

- **入口**：首页打开「Help & Support」聊天窗口并完成登录后，在窗口内 Header 下方出现「条数输入框 + 模拟对方连发」栏。
- **作用**：在**不经过 WebSocket** 的前提下，向当前会话的 store 中追加 N 条对方消息（Bot 或 Agent 身份），用于压测虚拟列表、滚动、大量消息下的 UI 表现。
- **实现**：`chatStore.simulateIncomingMessages(count)`：按当前 phase 生成 N 条 Message（content 为「模拟消息 1」…），push 到 `state.messages`。
- **文档**：详见 [智能客服IM-功能说明.md](./智能客服IM-功能说明.md) 第十节。

---

## 二、SSR 演示（传统 vs 流式）

- **路由**：`/demo/ssr-traditional`、`/demo/ssr-streaming`。
- **目的**：对比传统 SSR（等全量数据再输出 HTML）与流式 SSR（先壳 + Suspense 分块输出）的 TTFB、FCP、体验差异。
- **文档**：
  - [传统SSR与流式SSR对比.md](./传统SSR与流式SSR对比.md)：原理、指标、Suspense、流式 HTML。
  - [传统SSR与流式SSR-Next与React API对比.md](./传统SSR与流式SSR-Next与React API对比.md)：Next 与 React API 对照。
  - [流式SSR-Next与Node对比.md](./流式SSR-Next与Node对比.md)：Next 与 Node 实现方式对比。

---

## 三、test-ws（WebSocket 调试）

- **路由**：`/test-ws`。
- **作用**：连接真实 IM 后端（如 3001），展示连接状态、会话 ID、消息列表；可点击「对方发 N 条」请求服务端推送 Mock 消息（requestSimulatePush），用于验证长连接、批量消息、虚拟列表等。
- **与「模拟对方连发」区别**：test-ws 走真实 WebSocket 请求服务端推送；首页客服窗口内「模拟对方连发」为纯前端 store 注入，不经过 WS。

---

## 四、demo-protobuf（Protobuf 与分片）

- **路由**：`/demo-protobuf`。
- **作用**：连接后请求服务端一次性推送较多条消息（如 150 条），帧体可能超过 64KB，演示 Protobuf 编码及分片接收、解码与展示。
- **文档**：README 或 SDK 文档中「Protobuf + 大消息分片」小节。

---

## 五、stress（压测/高 QPS）

- **路由**：`/stress`。
- **作用**：高 QPS 或压力测试页面；可能涉及同源代理以避免浏览器扩展拦截跨域请求（如注释中提到的 Service Worker 拦截）。
- **前置**：需先打开聊天并连接（如从首页或 test-ws 连上）。

---

## 六、Server Actions 演示

- **路由**：`/demo/server-actions`。
- **作用**：演示 Next.js Server Actions 的调用方式与数据流。

---

## 七、AI 与 Stream

- **/ai**：AI 对话演示，流式输出、Markdown/代码块展示，可能调用 `/api/ai/chat`。
- **/stream**：流式内容演示页，用于展示流式渲染或流式数据加载的 UI 形态。
