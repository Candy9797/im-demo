# 聊天视频播放器实现说明

本文档说明聊天消息中「视频」的展示与播放实现，以及当前「播放不了」的常见原因与可行改法。

---

## 1. 实现方式概览

### 1.1 入口与数据流

- **消息类型**：`Message.type === MessageType.VIDEO` 时，按「视频消息」处理。
- **展示组件**：`MessageItem` 根据 `message.type` 渲染内容；当为 `VIDEO`（或 `IMAGE`/`PDF`）时，交给 **`FilePreview`** 渲染。
- **视频 URL 来源**：`message.content` 为主，若无则用 `message.metadata?.videoUrl`。

### 1.2 FilePreview 中的视频分支

**文件**：`src/components/FilePreview.tsx`

```tsx
if (message.type === MessageType.VIDEO) {
  const videoUrl = message.content || (message.metadata?.videoUrl as string);
  if (!videoUrl) return null;
  return (
    <div className="file-preview video-preview">
      <video
        src={videoUrl}
        controls
        playsInline
        preload="metadata"
        className="msg-video-player"
      >
        Your browser does not support the video tag.
      </video>
    </div>
  );
}
```

- 使用原生 **`<video>`**，`src` 即上面的 `videoUrl`。
- **controls**：显示浏览器自带进度条、音量、全屏等。
- **playsInline**：在移动端避免强制全屏。
- **preload="metadata"**：先只拉元数据，减少首屏流量。

### 1.3 样式

**文件**：`src/app/globals.css`

- `.video-preview`：容器最大宽度 320px、圆角、黑色底。
- `.msg-video-player`：块级、宽度 100%、最大高度 240px。

## 2. 视频 URL 从哪里来

### 2.1 Mock 预置（会话/群组）

**文件**：`src/lib/chatSessionMock.ts`

- 常量：`SAMPLE_VIDEO_URL = "https://commondatastorage.googleapis.com/gtv-videos-bucket/sample/BigBuckBunny.mp4"`（Google 公开样例）。
- 在「与 Alice 的 C2C」和「技术交流群」的预置消息里，会插入一条 `type: VIDEO`、`content: SAMPLE_VIDEO_URL` 的消息。

这类消息的 `content` 是**公网 HTTP(S) 地址**，不随刷新或重载丢失。

### 2.2 用户本地上传（发送视频）

**文件**：`src/store/chatSessionStore.ts` — `sendVideo(file)`

- 使用 **`URL.createObjectURL(file)`** 得到当前页面内的 **Blob URL**（形如 `blob:http://localhost:3000/xxx-xxx`）。
- 消息体里把该 Blob URL 写入 `content`，类型为 `VIDEO`，存入当前会话的 `messagesByConv`（仅内存 + 若存在 persist 则可能进 IndexedDB，但 **Blob 本身不会持久化**）。

因此：**用户发送的视频 = 以 Blob URL 形式存在，只在当前文档、当前会话生命周期内有效**。

## 3. 为什么现在播放不了？

常见原因有三类。

### 3.1 用户发送的视频：刷新/重开后失效（最主要）

- **原因**：Blob URL 指向的是浏览器内存里的 Blob。刷新、关掉页面或重新打开会话后：
  - 页面文档重建，原有 Blob 被回收；
  - 即使消息列表从 IndexedDB 或服务端恢复，`content` 里仍是旧的 `blob:...`，该 URL 已失效，`<video src="...">` 无法加载。
- **表现**：本地上传的视频在「当前会话未刷新」时能播，**刷新或重进会话后无法播放**。

**要能持久播放**，需要：上传到服务器（或 CDN），把返回的 **HTTP(S) 公网 URL** 写入 `message.content`（或 `metadata.videoUrl`），而不是存 Blob URL。当前实现没有上传步骤，因此用户发的视频只适合「当次会话内」播放。

### 3.2 Mock 样例地址（BigBuckBunny）无法加载

- **原因**：
  - 该 Google 存储桶可能调整策略、限制跨域或更换地址，导致在某些环境/地区无法访问；
  - 若页面是 HTTPS 而样例 URL 被重定向或证书异常，也可能被浏览器拦截；
  - 部分网络或防火墙会屏蔽部分外部域名。
- **表现**：预置的那条「样例视频消息」一直转圈或报错，无法播放。

**处理方式**：把 Mock 里的 `SAMPLE_VIDEO_URL` 换成你可控的、支持跨域和当前页面协议的公网视频地址（HTTPS 更稳妥），或先用本地静态资源（如 `/videos/sample.mp4`）做开发/演示。

### 3.3 其它（CORS、格式、策略）

- **CORS**：若视频来自其他域，服务器需返回正确的 `Access-Control-Allow-Origin` 等头，否则部分环境下可能无法播放或无法读取元数据。
- **格式/编码**：浏览器对编码有要求（如 H.264 + AAC 的 MP4 兼容性最好）；若文件格式或编码不被当前浏览器支持，也会播不了。
- **策略**：若页面在 iframe 或严格 CSP 下，可能限制 `video` 的 `src` 来源，需在策略里放行对应 URL。

## 4. 小结与建议

| 场景 | 实现要点 | 当前「播不了」的主要原因 |
|------|----------|---------------------------|
| Mock 预置视频 | `content` = 公网 URL（BigBuckBunny） | 样例 URL 可能失效/被墙/跨域或协议问题 |
| 用户发送视频 | `content` = `createObjectURL(file)` 的 Blob URL | Blob 未持久化，刷新或重进会话后 URL 失效 |

**建议**：

1. **用户上传视频**：增加「上传到后端/OSS」流程，消息里只存最终 **HTTPS URL**，不再存 Blob URL，这样刷新后仍可播放。
2. **Mock / 演示**：将 `SAMPLE_VIDEO_URL` 改为可靠可访问的地址（或本地静态文件），并确认该地址在目标环境下能直接在新标签页打开播放。
3. 若需保留「仅当次会话内」的 Blob 播放：可保持现状，但在 UI 上对「Blob URL 且已失效」的视频做降级展示（例如占位图 +「刷新后需重新发送」的提示）。
