# 引用回复与滚动聚焦信号

本文说明「回复」功能如何实现，以及 `scrollToInputRequest` 时间戳信号在其中的作用。

---

## 一、scrollToInputRequest 是什么

`scrollToInputRequest` 是 chatStore（以及 chatSessionStore）里的一个 **number 类型时间戳**，用作「请求滚动到底部并聚焦输入框」的**一次性信号**。

- **类型**：`number`（一般为 `Date.now()`）
- **含义**：每当需要「把消息列表滚到底部 + 让输入框获得焦点」时，就把这个值更新为一个新的时间戳；订阅该字段的组件通过 effect 检测变化后执行滚动/聚焦，并用 ref 记录「已处理过的值」避免重复执行。

### 为何用时间戳而不是布尔

若用 `boolean`（如 `true` 表示「需要滚动」），会出现：

- 用户连续两次触发「回复」：第一次设为 `true` 后，第二次再设 `true` 不会触发 effect（值没变），无法再次滚动/聚焦。
- 需要先设为 `false` 再设为 `true`，逻辑啰嗦且易出错。

用 **时间戳** 则每次触发都是新值，effect 每次都能响应；再用 **ref 存「上次已处理的时间戳」** 做去重，避免同一次信号被处理多次。

### 为何用 ref 存「上次已处理」而不用 state

- **不参与 UI**：这个值只用于 effect 里的比较（「当前信号是否已经处理过」），没有任何组件要把它渲染出来。用 state 存的话，每次 `setLastProcessed(xxx)` 都会触发一次重渲染，完全多余。
- **不触发 effect 依赖**：若用 state，effect 依赖里就要带上「上次已处理」；更新它又会改 state，容易把「何时执行滚动/聚焦」的逻辑搞乱。用 ref 只在 effect 内读/写 `lastScrollRequestRef.current`，不触发重渲染、也不进入依赖数组，只做本地簿记。
- **语义更合适**：ref 表示「组件内部需要跨渲染保留的引用/缓存」，state 表示「会影响渲染结果的数据」。这里「上次处理到哪个时间戳」明显是前者。
已在文档里加了一小节 「为何用 ref 存『上次已处理』而不用 state」，主要写了三点：
不参与 UI：这个值只给 effect 做比较用，没有地方要渲染它；用 state 会多一次无意义的 re-render。
不搅进 effect 依赖：用 state 就要把「上次已处理」放进依赖，更新它又会改 state，容易让「何时滚/何时聚焦」的逻辑变复杂；用 ref 只在 effect 里读写，不触发渲染、也不进依赖，只做本地簿记。
语义更合适：ref 表示「跨渲染保留的引用/缓存」，state 表示「影响渲染的数据」；「上次处理到哪个时间戳」属于前者。

## 二、谁在「写」scrollToInputRequest（触发方）

| 位置 | 时机 |
|------|------|
| **chatStore.replyToMessage(msg)** | 用户点击某条消息的「回复」按钮时：`set({ quoteTarget: msg, scrollToInputRequest: Date.now() })`，既设置引用目标，又发出「滚底 + 聚焦」信号。 |
| **chatStore.sendMessage 内部** | 发送成功后：`state.scrollToInputRequest = Date.now()`，发完消息后再触发一次滚底 + 聚焦，保证输入框仍在视口内、可继续输入。 |
| **chatStore.destroy()** | 销毁会话时重置：`scrollToInputRequest: 0`，清空信号。 |

chatSessionStore 中同理：`replyToMessage` 里 `set({ quoteTarget: msg, scrollToInputRequest: Date.now() })`。

---

## 三、谁在「读」scrollToInputRequest（响应方）

| 组件 | 作用 |
|------|------|
| **MessageList** | `useEffect` 依赖 `[scrollToInputRequest, messages.length]`。当 `scrollToInputRequest` 变化且与 `lastScrollRequestRef` 不同时，在 `setTimeout(0)` 后调用 `virtuosoRef.current?.scrollToIndex({ index: n - 1 })` 滚到底部，避免 Virtuoso 尚未用新数据渲染完就滚。 |
| **InputArea**（客服 IM） | `useEffect` 依赖 `[scrollToInputRequest]`。当 `scrollToInputRequest` 变化且未处理过时，`setTimeout(..., 100)` 后对输入框执行 `inputRef.current?.focus()`。 |
| **ChatSessionInput**（好友/群聊） | 同上：监听 `scrollToInputRequest`，在合适时机对输入框 `focus()`。 |

各组件用 **lastScrollRequestRef** 保存「已处理过的时间戳」，只有当前 `scrollToInputRequest !== lastScrollRequestRef.current` 才执行逻辑并更新 ref，避免同一次信号触发多次滚动或多次聚焦。

## 四、回复（引用回复）是怎么实现的

回复 = 「选中一条消息作为引用目标 + 滚到底部 + 聚焦输入框 + 在输入区展示引用预览 + 发送时把引用信息带在消息里」。

### 4.1 数据与 Store

- **quoteTarget**：当前要「回复」的那条消息（`Message | null`）。为 `null` 表示没有引用。
- **scrollToInputRequest**：时间戳信号，用于触发「滚底 + 聚焦」。
- **replyToMessage(msg)**：设置 `quoteTarget = msg` 且 `scrollToInputRequest = Date.now()`。
- **setQuoteTarget(msg | null)**：仅设置/清空引用目标（例如用户点「取消引用」）。

### 4.2 用户点击「回复」

1. **MessageItem** 每条消息有「回复」按钮，`onClick` 调用 `replyToMessage(message)`（客服 IM 用 chatStore，好友/群聊用 chatSessionStore 的 `replyToMessage`）。
2. Store 执行：`set({ quoteTarget: msg, scrollToInputRequest: Date.now() })`。
3. 订阅了 `quoteTarget` 和 `scrollToInputRequest` 的组件会更新。

### 4.3 列表滚底 + 输入框聚焦

- **MessageList** 的 effect 看到新的 `scrollToInputRequest`，执行 `scrollToIndex` 滚到底部。
- **InputArea / ChatSessionInput** 的 effect 看到新的 `scrollToInputRequest`，对输入框执行 `focus()`。

这样用户点「回复」后，列表会滚到底、输入框会获得焦点，便于直接输入。

### 4.4 引用预览（输入框上方）

- **QuotePreview**（客服 IM）和 **ChatSessionQuotePreview**（好友/群聊）从 store 读取 `quoteTarget`。
- 若 `quoteTarget` 非空，在输入框上方展示一条预览：发送者昵称 + 被引用内容截断（如 80 字）+ 「✕」取消按钮。
- 点击「✕」调用 `setQuoteTarget(null)`，仅清空引用，不触发滚动/聚焦。

### 4.5 发送时带上引用

- **sendMessage(content)** 在 store 里会读取当前 `quoteTarget`。
- 若存在，则构造 `metadata.quote`：
  - `messageId`、`senderName`、`content`（截断到 200 字）、`type`、`timestamp`。
- 调用 `client.sendMessage(content, MessageType.TEXT, metadata)`（或 chatSession 的等价发送），把引用信息随消息发出。
- 发送后立即 `set({ quoteTarget: null })`，清空引用并收起预览。

### 4.6 消息展示时的「引用块」

- 消息若带有 `metadata.quote`，在 **MessageItem** 等展示层会渲染「引用块」：被回复的那条消息的发送者 + 内容摘要，形成「回复 xxx：……」的视觉效果（具体 UI 由展示组件根据 `metadata.quote` 实现）。

## 五、流程串联（客服 IM 为例）

```
用户点击某条消息的「回复」
  → MessageItem 调用 replyToMessage(message)
  → chatStore set({ quoteTarget: message, scrollToInputRequest: Date.now() })

  → MessageList 的 effect 发现 scrollToInputRequest 变化
    → lastScrollRequestRef 去重后，setTimeout(0) 后 scrollToIndex(messages.length - 1) 滚到底

  → InputArea 的 effect 发现 scrollToInputRequest 变化
    → lastScrollRequestRef 去重后，setTimeout(100) 后 inputRef.current.focus() 聚焦

  → QuotePreview 因 quoteTarget 非空而渲染「引用预览」条（发送者 + 内容摘要 + 取消）

用户输入并点击发送
  → sendMessage(text)
  → store 用当前 quoteTarget 拼 metadata.quote，client.sendMessage(..., metadata)
  → set({ quoteTarget: null }) 清空引用
  → 可选：state.scrollToInputRequest = Date.now() 再触发一次滚底 + 聚焦
```
