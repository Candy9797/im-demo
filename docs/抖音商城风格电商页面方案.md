# 抖音商城风格电商页面方案

仿照抖音商城（Douyin Mall）交互与视觉风格，打造沉浸式、短平快的电商浏览体验。

---

## 一、产品定位

| 维度 | 说明 |
|------|------|
| **风格** | 全屏垂直滑动、单卡片沉浸、强视觉冲击 |
| **场景** | 短视频/直播种草后的快速转化、冲动消费 |
| **核心** | 内容即商品、滑动即浏览、所见即可买 |

---

## 二、技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **框架** | Next.js 16 App Router | 服务端组件、流式渲染、路由、Turbopack |
| **UI** | React 19 | `use()`、Suspense、并发渲染 |
| **性能** | 虚拟列表、IntersectionObserver、`content-visibility` | 长列表、懒加载、离屏优化 |
| **图片** | next/image | 自动 WebP、懒加载、占位 |
| **样式** | CSS Modules / 内联变量 | 隔离、树摇、主题一致 |
| **状态** | React 19 `useTransition` | 非阻塞交互 |
| **后端** | Node.js + Express（现有） | 可扩展 API |

## 四、核心交互

| 交互 | 实现 |
|------|------|
| **垂直滑动** | CSS `scroll-snap-type: y mandatory` + 每卡片 `scroll-snap-align: start` |
| **懒加载** | `IntersectionObserver` 仅加载可见/临近卡片内容 |
| **购买入口** | 每卡片右下角悬浮「立即购买」「加入购物车」 |
| **快速预览** | 点击卡片进入详情（可后续扩展） |

---

## 五、性能策略

| 策略 | 手段 | 代码位置 |
|------|------|----------|
| **虚拟化** | 仅渲染可见区 + 上下各 1 屏缓冲区 | `MessageList.tsx`（Virtuoso + OVERSCAN）、`HistoryMessageList.tsx`、`ChatSessionMain.tsx`（Virtuoso overscan） |
| **图片** | `loading="lazy"`、`placeholder="blur"`、尺寸适配 | `WaterfallCard.tsx`（Image loading/sizes）、`ProductCard.tsx`、`TaobaoProductCard.tsx`、`UrlPreviewCard.tsx`、`AIImage.tsx` |
| **离屏渲染** | `content-visibility: auto` 降低非可见 DOM 成本 | 待落地：可对列表项 `.wf-item` / 消息项加该属性 |
| **动画** | `transform` + `will-change`，避免 layout | `globals.css`（如 `slideUp`、弹窗/浮层动画均用 `transform`） |
| **代码分割** | 路由级 lazy、按需拆包 | Next.js App Router 路由级自动拆包；shop 页对商品列表（`/api/shop/products`）使用 `next/dynamic` 拆包 `ShopProductFeed`，其他页面不修改 |
| **并发更新** | `useTransition` 包裹非紧急状态更新 | `LoadMore.tsx`、`WaterfallList.tsx`、`ShopFeed.tsx` |

## 九、扩展方向

1. **详情页**：`/shop/[id]` 商品详情、规格选择、评价
2. **购物车**：Zustand 管理，与现有 IM 项目解耦
3. **搜索**：顶部搜索框跳转 `/shop/search`
4. **API**：对接 Express 商品列表、分类、推荐接口
5. **视频**：支持商品主图替换为短视频（HLS/MP4）
