# 转人工与排队流程

> 说明用户点击「Transfer to human support」后，为什么会有排队，以及系统做了哪些事情。

---

## 一、为什么有排队？

客服（Agent）数量有限，同一时间能接待的用户有限。当多个用户同时请求转人工时，需要**排队**按顺序分配。排队可以：

- 保证公平：先请求的先接入
- 避免超载：Agent 满员时新用户需等待
- 提供预期：展示排队位置、预估等待时间，降低用户焦虑

---

## 二、整体流程

```
用户点击「Transfer to human support」
    │
    ▼
IMClient.requestHumanAgent()
    │
    ├─ 发送 REQUEST_AGENT 帧到服务端
    ├─ 插入系统消息："Connecting you to a customer service representative..."
    └─ 设置 phase = QUEUING（进入排队阶段）
    │
    ▼
服务端 handleRequestAgent()
    │
    ├─ 创建 Agent 会话（createAgentConversation）
    ├─ 切换当前连接的会话 ID 到新 Agent 会话
    ├─ 初始化排队：position=3, total=5, estimatedWait=90s
    ├─ 发送 PHASE_CHANGE → 客户端进入 QUEUING
    └─ 启动定时器：每 2 秒 position-1，发送 QUEUE_STATUS
    │
    ▼
客户端收到 QUEUE_STATUS (queue_update)
    │
    └─ chatStore 更新 queue → QueueBanner 展示排队位置和预估时间
    │
    ▼
position 减到 0 时，服务端 assignAgent()
    │
    ├─ 分配客服（当前为固定 Agent #1024）
    ├─ 插入欢迎消息
    ├─ 发送 SESSION_SWITCHED
    └─ 更新 DB 会话 phase=agent
    │
    ▼
客户端收到 SESSION_SWITCHED
    │
    └─ 更新 conversation、messages，phase=AGENT → 进入人工对话
```

## 三、各环节具体动作

### 3.1 客户端（IMClient）

| 动作 | 代码位置 | 说明 |
|------|----------|------|
| 发送 REQUEST_AGENT | `IMClient.requestHumanAgent()` | WebSocket 发送 `{ type: "request_agent", payload: { conversationId, userId } }` |
| 插入系统消息 | 同上 | 本地插入 "Connecting you to a customer service representative..." |
| 设置 phase | 同上 | `setPhase(ConversationPhase.QUEUING)` |
| 监听 queue_update | `IMClient.setupEventListeners()` | 收到 `queue_status` 帧 → emit `QUEUE_UPDATE` |
| 监听 session_switched | 同上 | 收到后更新 conversation、messages，phase=AGENT |

### 3.2 服务端（ws-handler）

| 动作 | 代码位置 | 说明 |
|------|----------|------|
| 创建 Agent 会话 | `handleRequestAgent()` | `db.createAgentConversation(userId, parentBotId)`，新建 Agent 会话 |
| 切换连接会话 | 同上 | 将该用户所有连接的 `convId` 指向新 Agent 会话 |
| 初始化排队 | 同上 | DB 写入 `phase=queuing`, `queue_position=3`, `queue_total=5` |
| 发送 PHASE_CHANGE | 同上 | `{ phase: "queuing", queuePosition: 3, queueTotal: 5, estimatedWait: 90 }` |
| 定时更新排队 | 同上 | `setInterval` 每 2 秒 position-1，发送 `QUEUE_STATUS`，position≤0 时调用 `assignAgent()` |
| 分配客服 | `assignAgent()` | 固定分配 Agent #1024，插入欢迎消息，发送 `SESSION_SWITCHED` |

### 3.3 前端 UI（chatStore + QueueBanner）

| 动作 | 说明 |
|------|------|
| phase=QUEUING 时 | Header 显示 "Connecting to Agent..."，QueueBanner 展示排队信息 |
| queue 更新 | `client.on(SDKEvent.QUEUE_UPDATE, ...)` → `set({ queue })` |
| QueueBanner | 显示 `Queue position: #N`、`Estimated wait: ~Xs` 或 `~N min` |

---

## 四、当前实现特点（Mock）

- **排队为模拟**：服务端用 `setInterval` 每 2 秒 position-1（3→2→1→0），共约 6 秒
- **客服固定**：分配固定的 Agent #1024
- **预估时间**：`estimatedWait = position * 30` 秒

生产环境可改为对接真实客服系统，按实际空闲 Agent 和队列动态计算 position、estimatedWait。
