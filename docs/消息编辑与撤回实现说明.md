# 消息编辑与撤回实现说明

本文档说明本 IM 项目中消息编辑、撤回功能的完整实现流程，包括协议、服务端、客户端、Store 与 UI 各层。

---

## 一、功能概述

| 功能 | 条件 | 说明 |
|------|------|------|
| **编辑** | 本人消息、文本类型 | 修改消息内容，支持多端同步 |
| **撤回** | 本人消息、2 分钟内 | 将内容置为「已撤回」，并标记 `metadata.recalled` |

---

## 二、数据模型

### 2.1 编辑相关

- 编辑只修改 `content` 字段，不改变 `type`（仅 `text` 可编辑）
- 服务端直接覆盖 DB 中 `messages.content`

### 2.2 撤回相关

```typescript
// 撤回后的消息
{
  content: "已撤回",           // 显示文案
  type: "text",               // 统一为文本
  metadata: {
    recalled: true,           // 标记已撤回
    // 其他字段（如 reactions、readBy）保留
  }
}
```

---

## 三、WebSocket 协议

### 3.1 帧类型定义

```typescript
// src/sdk/types.ts - FrameType

// Client -> Server（客户端发送）
EDIT_MESSAGE: "edit_message",    // 编辑消息
RECALL_MESSAGE: "recall_message",// 撤回消息

// Server -> Client（服务端下发）
MESSAGE_EDIT: "message_edit",    // 编辑回执
MESSAGE_RECALL: "message_recall",// 撤回回执
```

### 3.2  payload 格式

| 帧类型 | 方向 | payload |
|--------|------|---------|
| `edit_message` | C2S | `{ messageId: string, content: string }` |
| `recall_message` | C2S | `{ messageId: string }` |
| `message_edit` | S2C | `{ messageId: string, content: string }` |
| `message_recall` | S2C | `{ messageId: string }` |
| `error`（撤回超时） | S2C | `{ code: "recall_expired", message: string, messageId: string }` |

## 四、服务端实现

### 4.1 DB 层

```typescript
// server/db.ts

// 新增：更新消息内容
export function updateMessageContent(msgId: string, content: string) {
  db.prepare("UPDATE messages SET content = ? WHERE id = ?").run(content, msgId);
}
```

已有 `updateMessageMetadata` 用于写入 `recalled` 等元数据。

### 4.2 多端广播：sendToConversation

编辑和撤回的 `MESSAGE_EDIT`、`MESSAGE_RECALL` 需发给会话所有参与者（用户+Agent），使用 `sendToConversation(convId, type, payload)` 替代 `sendToUser(userId, ...)`，内部根据 `db.getConversation(convId)` 获取 `user_id`、`agent_id` 并分别下发。

### 4.3 编辑流程 (EDIT_MESSAGE)

```
客户端发送 EDIT_MESSAGE
    ↓
1. 校验 messageId、content 非空
2. db.getMessage(messageId, convId) 获取消息
3. 校验：row.conversation_id === convId && row.sender_id === userId
4. 校验：row.msg_type === "text"（仅文本可编辑）
5. db.updateMessageContent(messageId, content.trim())
6. sendToConversation(convId, MESSAGE_EDIT, { messageId, content })  // 发给会话所有参与者（用户+Agent）
```

**代码位置**：`server/ws-handler.ts` → `case FrameType.EDIT_MESSAGE`

### 4.4 撤回流程 (RECALL_MESSAGE)

```
客户端发送 RECALL_MESSAGE
    ↓
1. 校验 messageId 非空
2. db.getMessage(messageId, convId) 获取消息
3. 校验：row.conversation_id === convId && row.sender_id === userId
4. 时间校验：Date.now() - row.timestamp <= 2 * 60 * 1000
   - 超时 → 发送 ERROR { code: "recall_expired", messageId }，直接返回
5. 更新 DB：
   - db.updateMessageContent(messageId, "已撤回")
   - db.updateMessageMetadata(messageId, JSON.stringify({ ...meta, recalled: true }))
6. sendToConversation(convId, MESSAGE_RECALL, { messageId })  // 发给会话所有参与者（用户+Agent）
```

**代码位置**：`server/ws-handler.ts` → `case FrameType.RECALL_MESSAGE`

### 4.5 权限与校验

| 校验项 | 编辑 | 撤回 |
|--------|------|------|
| 本人消息 | ✓ sender_id === userId | ✓ sender_id === userId |
| 同会话 | ✓ conversation_id === convId | ✓ conversation_id === convId |
| 消息类型 | 仅 text | 不限 |
| 时间限制 | 无 | 2 分钟内 |

## 五、客户端实现

### 5.1 IMClient (SDK 层)

**发送**：

```typescript
// src/sdk/IMClient.ts

editMessage(messageId: string, content: string): void {
  this.wsManager.send(FrameType.EDIT_MESSAGE, { messageId, content: content.trim() });
}

recallMessage(messageId: string): void {
  this.wsManager.send(FrameType.RECALL_MESSAGE, { messageId });
}
```

**事件转发**：WebSocketManager 收到 `MESSAGE_EDIT`、`MESSAGE_RECALL` 后，IMClient 转发为 `SDKEvent.MESSAGE_EDIT`、`SDKEvent.MESSAGE_RECALL`。

### 5.2 WebSocketManager

```typescript
// src/sdk/WebSocketManager.ts - handleFrame()

case FrameType.MESSAGE_EDIT:
  this.emit(SDKEvent.MESSAGE_EDIT, frame.payload);
  break;
case FrameType.MESSAGE_RECALL:
  this.emit(SDKEvent.MESSAGE_RECALL, frame.payload);
  break;
```

### 5.3 chatStore (状态层)

**编辑**：

```typescript
editMessage: (messageId: string, newContent: string) => {
  // 1. 乐观更新：直接修改 state.messages 中对应消息的 content
  set((state) => {
    const m = state.messages.find(x => x.id === messageId && x.senderId === auth.userId && x.type === MessageType.TEXT);
    if (m) m.content = newContent.trim();
  });
  // 2. 同步服务端
  client?.editMessage(messageId, newContent.trim());
}
```

**撤回**：

```typescript
recallMessage: (messageId: string) => {
  // 1. 记录原始内容，用于 recall_expired 时回滚
  const msg = get().messages.find(m => m.id === messageId && m.senderId === auth.userId);
  if (msg) _pendingRecallRevert = { messageId, content: msg.content };

  // 2. 乐观更新
  set((state) => {
    const m = state.messages.find(...);
    if (m) {
      m.content = "已撤回";
      m.type = MessageType.TEXT;
      m.metadata = { ...m.metadata, recalled: true };
    }
  });

  // 3. 同步服务端
  client?.recallMessage(messageId);
}
```

**事件订阅**（在 `initialize` 内，client 连接后）：

| 事件 | 处理逻辑 |
|------|----------|
| `MESSAGE_EDIT` | 更新对应消息的 `content`（多端同步 / 确认乐观更新） |
| `MESSAGE_RECALL` | 将对应消息置为「已撤回」并设置 `metadata.recalled`，同时清空 `_pendingRecallRevert` |
| `server_error` (code=recall_expired) | 用 `_pendingRecallRevert` 中的原始内容回滚乐观更新；并 `showToast(message)` 提示用户 |
| `server_error`（其他 code） | 若有 `payload.message`，`showToast(message)` 统一提示 |

### 5.4 MessageItem (UI 层)

**编辑**：

- 点击「编辑」→ `handleStartEdit`：进入编辑模式，`textarea` 展示当前内容
- 编辑后点击「确定」→ `handleConfirmEdit`：调用 `onEdit(message, trimmed)`
- 仅当 `message.type === MessageType.TEXT` 时显示编辑按钮

**撤回**：

- 2 分钟内显示「撤回」按钮：`canRecall = Date.now() - message.timestamp <= RECALL_LIMIT_MS`
- 点击「撤回」→ `handleRecall`：弹出自定义 `ConfirmModal` 二次确认，确认后调用 `onRecall(message)`
- 已撤回消息：`isRecalled = !!message.metadata?.recalled`，渲染为灰色「已撤回」

## 六、流程图

### 6.1 编辑流程

```
MessageItem（用户点击确定）
    ↓
onEdit(message, trimmed)
    ↓
chatStore.editMessage(messageId, newContent)
    ├── set() 乐观更新 content
    └── client.editMessage(messageId, newContent)
            ↓
        IMClient → wsManager.send(EDIT_MESSAGE, { messageId, content })
            ↓
        服务端 ws-handler
            ├── 校验 + db.updateMessageContent
            └── sendToConversation(convId, MESSAGE_EDIT, { messageId, content })
                    ↓
        WebSocketManager.handleFrame → emit(MESSAGE_EDIT, payload)
                    ↓
        chatStore 订阅 MESSAGE_EDIT → set() 更新 content（多端同步）
```

### 6.2 撤回流程（成功）

```
MessageItem（用户确认撤回）
    ↓
onRecall(message)
    ↓
chatStore.recallMessage(messageId)
    ├── _pendingRecallRevert = { messageId, content }
    ├── set() 乐观更新：content="已撤回", metadata.recalled=true
    └── client.recallMessage(messageId)
            ↓
        服务端：校验 + 更新 DB + sendToConversation(convId, MESSAGE_RECALL)
            ↓
        chatStore 订阅 MESSAGE_RECALL
            ├── 清空 _pendingRecallRevert
            └── set() 更新（通常已乐观更新，此处可做多端同步）
```

### 6.3 撤回流程（超时失败）

```
chatStore.recallMessage → 乐观更新 + client.recallMessage
    ↓
服务端：Date.now() - row.timestamp > 2min
    ↓
send(ws, ERROR, { code: "recall_expired", messageId })
    ↓
WebSocketManager → emit("server_error", payload)
    ↓
chatStore 订阅 server_error
    ├── 若 code === "recall_expired" && messageId 匹配 _pendingRecallRevert：
    │     set() 将 content 回滚为 _pendingRecallRevert.content，_pendingRecallRevert = null
    └── showToast(payload.message || "操作失败，请重试")
```

## 九、注意事项

1. **编辑仅支持文本**：图片、PDF、贴纸等类型不提供编辑入口。
2. **撤回时间**：客户端与服务端均校验 2 分钟，避免时钟偏差时依赖单一端。
3. **乐观更新**：编辑和撤回均先更新 UI，再同步服务端，提升响应速度。
4. **recall_expired 回滚**：使用 `_pendingRecallRevert` 记录撤回前内容，收到 `recall_expired` 时恢复原文并 Toast 提示「超过 2 分钟无法撤回」。
5. **多端同步**：`MESSAGE_EDIT`、`MESSAGE_RECALL` 通过 `sendToConversation` 发给会话所有参与者（用户+Agent），Agent 端也能实时看到编辑/撤回结果。
6. **自定义确认弹窗**：撤回操作使用 `ConfirmModal` 替代 `window.confirm`，风格与整体 UI 一致。
