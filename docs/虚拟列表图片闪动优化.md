# 虚拟列表滚动时图片闪动优化

## 问题背景

在使用虚拟列表（如 react-virtuoso）渲染 IM 消息时，快速滚动容易出现**图片闪动/白屏**。

### 为什么会闪？

虚拟列表为了性能，只渲染视口内（+ overscan 缓冲区）的 DOM 节点，视口外的节点会被卸载。当用户快速滚动时：

1. **卸载 → 重新挂载**：图片所在的消息项从视口外进入视口，React 重新挂载组件、创建新的 `<img>` 元素，浏览器需要重新发起请求（即使有缓存也需要解码），导致先白后显的闪动。
2. **无预留空间 → 布局抖动**：如果图片没有预留固定宽高，加载前容器高度为 0，加载完成后突然撑高，Virtuoso 重新计算 scrollHeight，导致整个列表抖动甚至跳位。
3. **同步解码阻塞**：默认情况下浏览器在主线程同步解码图片，快速滚动时多张图片同时进入视口，解码会阻塞渲染，出现卡顿或帧间闪白。

### 体感上的表现

- 快速向上/向下滚动时，图片位置先是一块空白/灰色，然后突然「闪」出图片。
- 图片加载完成后列表跳动，滚动位置偏移，用户感觉被"弹"了一下。
- 滚动过程中帧率下降，出现卡顿感。

---

## 解决方案

### 1. 预留固定空间（最关键）

给图片容器设定固定的宽高比或最小高度，让 Virtuoso 在计算每行高度时，无论图片是否加载，容器尺寸不变。

```tsx
<div
  className="image-preview"
  style={{
    aspectRatio: '4/3',
    minHeight: 80,
    maxWidth: 220,
    position: 'relative',
    backgroundColor: '#1e2329',
  }}
>
  <img ... style={{ position: 'absolute', inset: 0, width: '100%', height: '100%', objectFit: 'contain' }} />
</div>
```

**原理**：容器有固定尺寸 → Virtuoso 测量到的行高稳定 → 加载前后 scrollHeight 不变 → 不抖动。

### 2. 占位 + 加载后淡入

加载前展示同色/骨架占位，加载完成后用 opacity 淡入，避免「突然出现」。

```tsx
const [loaded, setLoaded] = useState(false);

{/* 占位层 */}
{!loaded && <div style={{ position: 'absolute', inset: 0, background: '#1e2329' }} />}

{/* 图片 */}
<img
  onLoad={() => setLoaded(true)}
  style={{ opacity: loaded ? 1 : 0, transition: 'opacity 0.15s ease-out' }}
/>
```

**原理**：加载期间视觉上是一块稳定色块，加载完 opacity 从 0 过渡到 1，用户感知是「平滑出现」而非「闪一下」。

### 3. `decoding="async"`

```tsx
<img decoding="async" ... />
```

**原理**：告诉浏览器图片解码可以在后台线程做，不阻塞主线程的渲染和滚动。对快速滚动时多张图片同时进入视口的场景效果明显，减少卡顿。

### 4. `loading="lazy"`

```tsx
<img loading="lazy" ... />
```

**原理**：浏览器原生懒加载，只有图片接近视口时才发起请求。减少页面初始化时的并发请求数，避免不可见图片抢带宽和解码资源。

### 5. 调大 overscan

```tsx
<Virtuoso overscan={5} ... />
```

**原理**：视口外多渲染几条消息，这些消息里的图片提前挂载和加载，用户滚到时已经加载完成，不再闪动。代价是多渲染几个 DOM 节点，可根据体感在 3~10 间调整。

### 6. 进阶：缩略图 / 模糊占位（blurHash）

如果服务端支持，可以在消息 metadata 里带上：

- **缩略图 URL**：先展示小图，点击/进入视口后换大图。
- **blurHash**：一个很短的字符串，可以解码为模糊色块图，作为占位。加载完成后替换为真实图片。
- **宽高信息**：`metadata.width` / `metadata.height`，容器可以用真实宽高比预留空间，比固定 4:3 更精确。

```tsx
// 假设 metadata 里带了宽高
const w = message.metadata?.width as number || 220;
const h = message.metadata?.height as number || 165;

<div style={{ aspectRatio: `${w}/${h}`, maxWidth: 220 }}>
  <img src={message.content} ... />
</div>
```

| `object-fit: contain` | `FilePreview.tsx` | 图片贴合容器，不拉伸不裁剪 |

## 面试回答版（一段话）

虚拟列表滚动时图片闪动，根本原因是虚拟列表只渲染视口内的节点，图片进入视口时才挂载和加载，加载前没有预留空间就会布局抖动，加载瞬间「突然出现」就是视觉上的闪动。我们的做法：第一，给图片容器设固定宽高比和最小高度，不管图片是否加载完，容器尺寸不变，Virtuoso 计算行高就稳定，不会抖；第二，加载前用同色占位、加载完用 opacity 淡入，用户感知是「平滑出现」而不是「闪一下」；第三，img 标签加 `decoding="async"`，让图片解码不阻塞主线程的滚动渲染；第四，overscan 设为 5，视口外多渲染几条，图片提前加载，滚到时已经好了。如果服务端支持，还可以带上缩略图或 blurHash 做模糊占位，以及消息里带图片宽高信息做精确占位。
