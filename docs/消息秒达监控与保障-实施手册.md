## 一、监控埋点接入

### 1.1 客户端埋点（IMClient / MessageQueue）

在 `src/sdk/IMClient.ts` 或封装层增加：

```typescript
// 发送时记录
const sendTs = Date.now();
const msgWithTs = { ...msg, _sendTs: sendTs };
this.wsManager.send(FrameType.SEND_MESSAGE, msgWithTs);

// message_ack 回调中
this.wsManager.on("message_ack", (payload: unknown) => {
  const p = payload as { _sendTs?: number; clientMsgId?: string };
  if (p._sendTs) {
    const latency = Date.now() - p._sendTs;
    console.log("[IM] send_to_ack_ms", latency);
    // 可选：上报到监控系统
    // reportMetric("im.send_to_ack_ms", latency, { conversationId });
  }
  // ... 原有 ACK 处理逻辑
});
```

**注意**：服务端需透传 payload，勿丢弃 `_sendTs` 等扩展字段。

### 1.2 服务端埋点（ws-handler）

在 `server/ws-handler.ts` 的 `handleFrame` 中：

```typescript
case FrameType.SEND_MESSAGE: {
  const recvTs = Date.now();
  // ... 原有处理 ...
  const serverLatency = Date.now() - recvTs;
  if (serverLatency > 200) {
    console.warn("[WS] slow SEND_MESSAGE", { serverLatency, convId: convId });
  }
  // 可选：在 ACK payload 中带回 _serverLatency 供客户端展示
  sendToUser(userId, FrameType.MESSAGE_ACK, { ...ackPayload, _serverLatency: serverLatency });
  break;
}
```

## 二、可观测性建设

### 2.1 日志规范

| 场景 | 日志级别 | 内容示例 |
|------|----------|----------|
| 发送 | DEBUG | `[IM] send msgId=${msg.id} convId=${convId}` |
| ACK | INFO | `[IM] ack msgId=${msgId} latency=${latency}ms` |
| 慢消息 | WARN | `[IM] slow_ack msgId=${msgId} latency=${latency}ms` |
| 连接断开 | WARN | `[WS] disconnect userId=${userId} reason=${reason}` |

### 2.2 指标上报（可选）

若接入 Prometheus / Datadog 等：

- `im_send_to_ack_ms`：直方图，标签 `conversation_id`
- `im_message_received_total`：计数器
- `im_ws_connections`：当前连接数

### 2.3 简易监控面板

- 发送到 ACK 平均耗时、P99
- 消息接收 TPS
- WebSocket 连接数、断线次数

## 三、参数调优建议

| 目标 | 调整方向 |
|------|----------|
| 降低 ACK 延迟 | 将 MessageQueue `flushInterval` 调小至 50ms |
| 降低服务端负载 | 将 `flushInterval` 调大至 150ms，`batchSize` 调大 |
| 降低 IndexedDB 压力 | 增大 `WRITE_DEBOUNCE_MS`、`MAX_BUFFER_SIZE` |
| 降低无效连接 | 调大 `heartbeatInterval`（如 60s）|
