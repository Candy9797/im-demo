# 弱网环境测试说明

> 本项目未内置弱网模拟，可通过浏览器/系统/代理工具或手动断网来验证重连、SYNC 补拉、IndexedDB 恢复等行为。

---

## 一、Chrome DevTools 限速（最方便）

1. 打开 **Chrome DevTools** → **Network** 面板。
2. 在 **Throttling** 下拉框中选择：
   - **Slow 3G**：高延迟、低带宽，可观察消息延迟到达、重连、SYNC 补拉。
   - **Fast 3G**：中等延迟，可观察批处理、去重在延迟下的表现。
   - **Offline**：模拟断网，连接会断开，恢复后应触发重连、网络恢复时立即重连（`window.online`）。
3. 建议在 **/test-ws** 或 **/chat** 页连接后切换限速/离线，观察：
   - 连接状态是否变为 RECONNECTING / DISCONNECTED；
   - 恢复网络后是否自动重连并收到 SYNC 补拉；
   - 刷新或断网期间发送的消息在恢复后是否通过 ACK/重试或 SYNC 补全。

**注意**：Chrome 的 Throttling 主要作用于 HTTP 请求；WebSocket 的建连和帧传输也会受一定影响，但不如系统级限速彻底。

---

## 二、系统级限速（更真实）

### macOS：Network Link Conditioner

1. 安装：**Xcode** → **Additional Tools** → **Network Link Conditioner**，或在 [Apple 开发者下载](https://developer.apple.com/download/all/) 搜索 “Hardware IO Tools” 安装。
2. 系统设置 → **开发者** → **Network Link Conditioner**，开启 Profile（如 **100% Loss**、**3G**、**Very Bad Network**）可模拟丢包、高延迟、低带宽。
3. 对本机所有流量生效，WebSocket 与 HTTP 都会受影响，适合验证重连、Pong 超时、指数退避。

### Windows

- 可用 **Fiddler**、**Clumsy** 等工具做延迟/丢包；或 **Windows 设置** 中部分版本有“计量连接”等限速选项。

## 三、代理工具（可细调）

- **Charles** / **Fiddler**：可对域名做 Throttling（延迟、带宽），对 WebSocket 连接建立和后续流量生效。
- 配置代理后，在工具里为该 IM 服务（如 `127.0.0.1:3001`、`localhost:3000`）启用限速或断线，观察连接状态与重连。

---

## 四、手动断网 / 停服务（验证重连与恢复）

| 操作 | 预期现象 |
|------|----------|
| **关闭 WiFi / 拔网线** | 连接进入 RECONNECTING，指数退避重连；恢复网络后应触发 `window.online` 立即重连（见 `WebSocketManager.bindPageLifecycle`）。 |
| **停止 IM 服务端**（如 `Ctrl+C` 停 `tsx server/index.ts`） | 客户端 onclose → RECONNECTING，按退避间隔重试；重启服务后应连上并收到 auth_ok，必要时 SYNC 补拉。 |
| **刷新页面** | 连接重建；若 auth_ok 前有 IndexedDB 持久化，可先展示本地历史，再等 auth_ok/sync 补全。 |

---

## 五、本项目在弱网下的设计（可验证点）

- **心跳与 Pong 超时**：30s Ping，10s 内未收到 Pong 则主动断开并重连，可应对半开连接、NAT 超时。
- **指数退避重连**：`delay = min(1000×2^n + jitter, 30000)`，最多 5 次；网络恢复时立即重连并重置次数。
- **SYNC 补拉**：重连成功后发 `SYNC { afterSeqId }`，服务端返回断线期间消息，与本地合并、去重、按 seqId 排序。
- **IndexedDB**：收到消息写入本地，断网/刷新后可先读本地再等 sync；auth_ok 空消息时也会用本地数据展示。
- **MessageQueue**：ACK 超时重发、断线 pendingAck 回滚，弱网下易触发重试与回滚，可观察发送状态与去重。

## 六、建议测试流程（示例）

1. 打开 **/test-ws**，连接并发几条消息。
2. DevTools → Network → **Offline**，观察状态变为 DISCONNECTED/RECONNECTING，再发一条（应进待发队列或失败提示）。
3. 切回 **Online**，观察是否自动重连、是否收到 SYNC 补拉、列表是否完整无重。
4. 可选：用 **Network Link Conditioner** 开 100% Loss 或 Very Bad Network，再重复 2～3，验证退避与恢复。
