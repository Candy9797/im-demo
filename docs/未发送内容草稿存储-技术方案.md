# 未发送内容草稿存储 - 技术方案

本文档说明「未发送内容存 IndexedDB、页面恢复后可继续编辑或发送」的实现方式与存储结构。

---

## 1. 目标与范围

### 1.1 功能目标

- 用户在输入框中输入但**未点击发送**时，将当前内容持久化到本地（IndexedDB）。
- 用户**刷新页面、关闭后重新打开、或切换会话再切回**时，从本地恢复该场景下的草稿到输入框，并提示「已恢复未发送内容，可继续编辑或发送」。
- 用户**点击发送**后，清除对应草稿，避免下次恢复出已发内容。

### 1.2 适用场景

| 场景           | 入口 / 组件              | 草稿 key 约定           |
|----------------|--------------------------|--------------------------|
| 好友 / 群聊会话 | `/chat`，`ChatSessionInput` | `chat:${conversationId}` |
| 智能 / 人工客服 | 首页客服浮窗，`InputArea`   | 固定 `customer-service`  |

---

## 2. 存储结构（IndexedDB）

### 2.1 库与表

- **数据库名**：`im-demo-drafts`（与聊天消息持久化库 `im-demo-chat` 分离，避免混用）。
- **Object Store 名**：`drafts`。
- **主键**：`id`（keyPath），类型为字符串，即草稿的唯一标识（见下文 key 约定）。

### 2.2 单条记录结构（DraftRecord）

每条草稿对应一条记录，主键为 `id`，值为一个对象：

```ts
interface DraftRecord {
  id: string;      // 草稿 key，与 Object Store 主键一致
  text: string;    // 未发送的纯文本内容
  updatedAt: number; // 最后更新时间戳，Date.now()
}
```

### 2.3 Key 的约定（id 取值）

| id 值 | 含义 | 使用处 |
|-------|------|--------|
| `chat:${conversationId}` | 某会话的输入草稿 | `ChatSessionInput`，如 `chat:group-1`、`chat:friend-2` |
| `customer-service` | 客服浮窗输入草稿（智能/人工共用） | `InputArea` |

同一 id 只会有一条记录；`setDraft(id, text)` 使用 `put` 覆盖，`getDraft(id)` 按 id 读取，`clearDraft(id)` 按 id 删除。

### 2.4 结构示意

```
IndexedDB
└── im-demo-drafts (version 1)
    └── drafts (Object Store, keyPath: "id")
        ├── { id: "chat:group-1",  text: "你好…", updatedAt: 1739123456789 }
        ├── { id: "chat:friend-2", text: "在吗",   updatedAt: 1739123460000 }
        └── { id: "customer-service", text: "我要转人工", updatedAt: 1739123500000 }
```

## 3. 实现方式

### 3.1 草稿读写层：draftStorage

**文件**：`src/lib/draftStorage.ts`

- **getDraft(id: string): Promise<string | null>**  
  按 id 读取，返回 `record.text`；无记录或无 text 则返回 `null`。
- **setDraft(id: string, text: string): Promise<void>**  
  写入或覆盖：`put({ id, text, updatedAt: Date.now() })`。
- **clearDraft(id: string): Promise<void>**  
  发送成功后删除该草稿：`delete(id)`。

每次读写都会 `openDB()`，在事务完成后 `db.close()`，无全局长连接；防抖由调用方在 UI 层做（见下）。

### 3.2 会话页输入：ChatSessionInput

**草稿 key**：`draftId = activeConversation ? \`chat:${activeConversation.id}\` : null`。

- **与 ref 同步**  
  `inputTextRef.current = inputText`，保证防抖回调和切换会话时拿到的是当前输入的最新值。
- **切换会话**  
  - 当 `draftId` 变化时（如从 A 切到 B）：  
    - 若有「上一会话」`prevId` 且 `prevId !== draftId`，先把当前 `inputTextRef.current` 保存到 `prevId`（`setDraft(prevId, ...)`）。  
    - 再对**新** `draftId` 执行 `getDraft(draftId)`，用返回的文本 `setInputText`，若有非空内容则 `setRestoredFromDraft(true)` 显示恢复提示。
  - 用 `prevDraftIdRef` 记录「上一轮」的 draftId，以便在 effect 里区分「从 A 切到 B」。
- **防抖写入**  
  依赖 `[draftId, inputText]` 的 effect，500ms 后执行：若 `inputTextRef.current.trim()` 非空则 `setDraft(draftId, text)`，否则 `clearDraft(draftId)`。通过 ref 取最新内容，避免闭包拿到旧值。
- **发送**  
  `handleSend` 里在 `sendMessage` 后执行 `clearDraft(draftId)`，并 `setRestoredFromDraft(false)`。

### 3.3 客服输入：InputArea

**草稿 key**：固定 `DRAFT_ID_CUSTOMER_SERVICE = 'customer-service'`。

- **挂载恢复**  
  仅执行一次的 effect（`[]`）中调用 `getDraft(DRAFT_ID_CUSTOMER_SERVICE)`，若有非空字符串则 `setText(saved)` 并 `setRestoredFromDraft(true)`。
- **防抖写入**  
  依赖 `[text]` 的 effect，500ms 后根据 `textRef.current.trim()` 决定 `setDraft` 或 `clearDraft`。
- **发送**  
  `handleSend` 里在 `sendMessage` 后执行 `clearDraft(DRAFT_ID_CUSTOMER_SERVICE)`，并 `setRestoredFromDraft(false)`。

### 3.4 防抖与 ref 的用意

- **防抖 500ms**  
  输入过程中不每次按键都写 IndexedDB，减少 I/O 和卡顿；松手/切会话前最后一次内容会在下一次输入或切会话时被写入（或由「切换会话」逻辑立刻写入旧会话）。
- **用 ref 取最新内容**  
  防抖回调在 500ms 后执行，若直接用 state 闭包可能拿到旧值；通过 `inputTextRef.current` / `textRef.current` 始终读到当前输入，保证写入内容与界面一致。

---

## 4. 数据流简述

- **输入中**：用户输入 → state 更新 → ref 同步 → 500ms 后 `setDraft(id, text)`（有内容）或 `clearDraft(id)`（清空）。
- **切换会话（仅 ChatSessionInput）**：`draftId` 变化 → 先 `setDraft(prevId, inputTextRef.current)` 保存旧会话 → 再 `getDraft(draftId)` → `setInputText(loaded)`，若有内容则展示「已恢复未发送内容」。
- **刷新 / 重开**：组件重新挂载 → `getDraft(id)`（会话页随当前 `activeConversation` 的 id，客服用固定 id）→ 有则回填输入框并展示恢复提示。
- **发送**：`sendMessage` 后对该 id 执行 `clearDraft(id)`，并关闭恢复提示。

---

## 6. 存储结构小结

- **库**：`im-demo-drafts`，单 store `drafts`，主键 `id`（字符串）。
- **记录**：`{ id, text, updatedAt }`，仅文本草稿，无富文本/附件。
- **key 约定**：会话页 `chat:${conversationId}`，客服 `customer-service`；同一 id 唯一一条记录，覆盖写入，发送后删除。
