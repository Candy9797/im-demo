# 聊天会话滚动位置保持 - 技术说明

## 一、功能概述

在好友/群聊会话中，切换对话（例如从「Alice」切到「项目协作」再切回「Alice」）时，**每个会话的滚动位置会被记住并恢复**，避免每次切回来都回到底部或顶部，提升连续阅读体验。

- **适用范围**：单聊（C2C）、群聊（Group）共用同一套逻辑。
- **存储方式**：仅内存（`useRef`），不持久化；刷新页面后进入会话仍从底部开始。

---

## 二、会话标识：conversationKey

每个会话用 `conversationKey` 唯一标识，用于读写「该会话上次的滚动位置」：

| 类型 | conversationKey 规则 | 示例 |
|------|----------------------|------|
| 单聊 | `getConversationKey('c2c', friendId)` | `c2c:user-alice` |
| 群聊 | `activeConversation.id`（群 ID） | `group-1` |

实现位置：`ChatSessionMain` 中通过 `activeConversation.type` 与 `activeConversation.id` 计算得出。

---

## 三、存储结构

- **变量**：`scrollTopIndexByKeyRef = useRef<Record<string, number>>({})`
- **含义**：`key` 为 `conversationKey`，`value` 为该会话**当前可见区域顶部对应的消息在列表中的 index**（即 Virtuoso 的「顶部可见条目的下标」）。
- **更新时机**：Virtuoso 的 `rangeChanged` 回调在用户滚动或列表变化时触发，将 `range.startIndex` 写入 `scrollTopIndexByKeyRef.current[conversationKey]`。
- **不持久化**：刷新或关闭页面后该 ref 清空，再次进入会话时按「无记录」处理，滚到底部。

---

## 四、目标滚动下标：scrollToIndex

进入某会话时，要滚到的下标 `scrollToIndex` 计算规则：

1. 若存在该会话的已存下标 `saved`，且 `0 <= saved < messages.length`，则 `scrollToIndex = saved`。
2. 否则（首次进入、刷新后、或存的下标已越界）：`scrollToIndex = Math.max(0, messages.length - 1)`，即**滚到底部**。

这样既能在切换回来时恢复位置，又能在新会话或刷新后保持「从底部开始」的常规体验。

---

## 五、为何仅用 initialTopMostItemIndex 不够

Virtuoso 的 `initialTopMostItemIndex` 只在**组件首次挂载**时生效，用来设定初始滚动位置。

- **单聊/群聊共用一个 Virtuoso 实例**：切换会话时并不会卸载 Virtuoso，只是更新其 `data`（消息列表）和 `initialTopMostItemIndex`。
- **切换会话后**：Virtuoso 内部不会因为 `initialTopMostItemIndex` 的 props 变化而重新执行「初始滚动」逻辑，因此**群聊（以及单聊）切回来时，滚动位置不会恢复**，表现像是没有「记住」位置。

因此需要**在会话切换后主动调用一次滚动 API**，才能恢复之前保存的位置。

---

## 六、恢复方式：ref + scrollToIndex

1. **Virtuoso 的 ref**  
   使用 `VirtuosoHandle` 类型的 ref，在会话切换、列表更新后调用其 `scrollToIndex` 方法。

2. **恢复时机**  
   在 `useEffect` 中依赖 `conversationKey`、`scrollToIndex`、`messages.length`：当切换会话或当前会话消息数量变化时，在下一个事件循环（`setTimeout(0)`）中执行一次 `virtuosoRef.current?.scrollToIndex({ index: scrollToIndex, align: 'start', behavior: 'auto' })`。  
   延后一帧是为了在 Virtuoso 已用新 `data` 完成一帧渲染后再滚动，避免拿到旧列表或布局未完成导致位置不准。

3. **仍保留 initialTopMostItemIndex**  
   首次进入页面、Virtuoso 第一次挂载时，仍通过 `initialTopMostItemIndex={scrollToIndex}` 滚到正确位置（底部或上次保存的位置）。

4. **rangeChanged 持续写入**  
   用户滚动时，`rangeChanged` 继续把当前 `range.startIndex` 写入 `scrollTopIndexByKeyRef.current[conversationKey]`，供下次切回该会话时使用。

---

## 七、数据流小结

```
用户滚动 / 列表变化
  → rangeChanged(range) 
  → scrollTopIndexByKeyRef.current[conversationKey] = range.startIndex

用户切换会话（或同一会话消息数变化）
  → conversationKey / messages 变化 
  → scrollToIndex 重新计算（有存用存，否则底）
  → useEffect 触发 
  → setTimeout(0) 后 virtuosoRef.current.scrollToIndex({ index: scrollToIndex, ... })
  → 列表滚到目标位置
```

## 九、面试要点（可选）

- **问题**：多会话切换时如何保持每个会话的滚动位置？
- **要点**：按会话 key 存「顶部可见条 index」；Virtuoso 的 `initialTopMostItemIndex` 仅首挂载生效，切换会话后需用 ref 调 `scrollToIndex` 主动恢复；用 `rangeChanged` 实时把 `range.startIndex` 写回内存；不持久化则刷新后从底部开始。
