# 按住说话（语音输入）技术说明

本文档说明项目中「按住说话」语音输入功能的用途、技术原理与实现细节。

---

## 1. 功能概述

### 1.1 做什么

- **按住说话**：用户按住麦克风按钮时开始语音识别，松手后结束识别，并将识别出的文字**填入当前输入框**（可继续编辑或发送）。
- **实时反馈**：按住期间，识别引擎会持续返回**中间结果**（interim results），页面上以「正在识别：xxx」的形式实时展示，边说边出字。
- **无侵入**：识别结果仅追加到输入框，不自动发送；用户可修改后再发送。

### 1.2 使用场景与入口

| 场景           | 位置                     | 组件 / 路由说明                          |
|----------------|--------------------------|------------------------------------------|
| 好友/群聊会话  | 聊天页 `/chat` 输入区    | `ChatSessionInput`（会话页主输入）       |
| 智能/人工客服  | 首页客服浮窗内输入区     | `InputArea`（ChatWindow 内，Bot/Agent 共用） |

两处均复用同一套：**Hook `useSpeechRecognition` + 组件 `HoldToTalkButton`**，仅集成方式（如占位符、`disabled` 条件）不同。

---

## 2. 技术原理

### 2.1 依赖：Web Speech API（语音识别）

功能完全基于浏览器标准 **Web Speech API** 的 **语音识别（Speech Recognition）** 部分：

- **规范**：W3C 草案 [Web Speech API](https://w3c.github.io/speech-api/)（非所有浏览器都实现完整）。
- **能力**：将麦克风采集的语音流，在浏览器端或云端转为文本（具体实现因浏览器/厂商而异）。
- **无需自建模型**：不依赖项目内 ASR 服务，只要浏览器支持即可使用。

### 2.2 核心对象：`SpeechRecognition`

- **构造**：`new SpeechRecognition()` 或 `new webkitSpeechRecognition()`（部分浏览器仍带 `webkit` 前缀）。
- **流程**：`start()` 开始监听麦克风 → 引擎持续识别 → 通过事件回传结果 → `stop()` 或 `abort()` 结束。

### 2.3 关键参数

| 属性 / 方法     | 说明 |
|-----------------|------|
| `lang`          | 识别语言，如 `zh-CN`、`en-US`。本项目默认 `zh-CN`。 |
| `continuous`    | `true` 时在单次 `start()` 到 `stop()` 之间持续识别多段话；`false` 则说一句就停。本项目为 `true`，以支持「按住期间连续说」。 |
| `interimResults`| `true` 时除最终结果外还会返回中间结果（边说边出的不完整文本）。本项目为 `true`，用于实时展示「正在识别：xxx」。 |
| `start()`       | 开始识别；会请求麦克风权限（若尚未授权）。 |
| `stop()`        | 结束识别，但会尽量把**已捕获的语音**做一次最终识别并触发 `onresult` + `onend`，**适合「松手」时调用**。 |
| `abort()`       | 立即终止识别，**不**尝试返回结果，适合组件卸载等「不需要结果」的清理。 |

### 2.4 为什么松手要用 `stop()` 而不是 `abort()`

- **`abort()`**：直接终止，不返回识别结果；用户松手后拿不到文字。
- **`stop()`**：结束监听，并尝试对当前已采集的语音做最终识别，再触发 `onresult`（可能带最后一段的 final 结果）和 `onend`，因此能在松手时拿到完整识别文本。

因此用户松手时必须调用 `rec.stop()`，在 `onend` 中再执行清理和回调；仅在卸载等场景使用 `rec.abort()`。

### 2.5 事件与结果格式

- **`onresult`**：每次有识别结果时触发。  
  - `event.results`：结果列表。  
  - 每项有 `isFinal`（是否最终结果）和 `transcript`（文本）。  
  - `interimResults === true` 时会有多段 `isFinal === false` 的中间结果，再跟一段 `isFinal === true` 的最终结果。
- **`onend`**：识别会话结束（`stop()` 或 `abort()` 导致，或异常结束）。
- **`onerror`**：错误，如 `not-allowed`（未授权麦克风/用户拒绝）、`no-speech` 等。

## 3. 实现架构

### 3.1 分层与职责

```
┌─────────────────────────────────────────────────────────────┐
│  输入场景（ChatSessionInput / InputArea）                     │
│  - 状态：inputText、voiceInterim                              │
│  - 展示：「正在识别」文案、输入框、发送按钮                     │
└───────────────────────────┬─────────────────────────────────┘
                            │ onResult(text) / onInterim(text) / onEnd()
┌───────────────────────────▼─────────────────────────────────┐
│  HoldToTalkButton                                             │
│  - 指针：按下 start()、松手/移出 stop()                        │
│  - 将 Hook 的 final/interim/end 映射为 onResult/onInterim/onEnd │
└───────────────────────────┬─────────────────────────────────┘
                            │ useSpeechRecognition()
┌───────────────────────────▼─────────────────────────────────┐
│  useSpeechRecognition（Hook）                                 │
│  - 创建/持有 SpeechRecognition 实例                           │
│  - continuous + interimResults，收集 final 段到 ref            │
│  - 暴露 start/stop/getFinalTranscript、isListening、error    │
└───────────────────────────┬─────────────────────────────────┘
                            │ 浏览器 API
┌───────────────────────────▼─────────────────────────────────┐
│  SpeechRecognition / webkitSpeechRecognition                 │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 Hook：`useSpeechRecognition`

- **文件**：`src/hooks/useSpeechRecognition.ts`
- **作用**：封装 Web Speech API，管理识别生命周期与结果汇总。
- **配置**：`lang`、`continuous`、`interimResults`，以及回调 `onResult(transcript, isFinal)`、`onEnd`、`onError`。
- **状态**：`isListening`、`error`；内部用 `finalTranscriptRef` 累积本次「按住期间」所有 `isFinal === true` 的文本。
- **返回值**：`supported`、`isListening`、`error`、`start`、`stop`、`getFinalTranscript()`。  
  - 松手后由调用方在 `onEnd` 里调用 `getFinalTranscript()` 得到本次完整文本并写入输入框。

### 3.3 组件：`HoldToTalkButton`

- **文件**：`src/components/HoldToTalkButton.tsx`
- **作用**：将「按住/松手」的指针事件映射为 `start()`/`stop()`，并把识别结果通过 props 回传给父组件。
- **交互**：
  - **按下**（`onPointerDown`）：`start()`，开始识别。
  - **松手**（`onPointerUp`）或**指针移出**（`onPointerLeave` 且仍按住）：`stop()`，结束识别。
  - **右键**：`onContextMenu` 阻止，避免与按住冲突。
- **Props**：`lang`、`onResult(text)`、`onInterim(text)`、`onEnd()`、`holdTitle`、`unsupportedTitle`、`disabled` 等。  
  - `onInterim`：实时中间结果，用于展示「正在识别：xxx」。  
  - `onEnd`：识别结束（含松手），用于清空「正在识别」等 UI。  
  - `onResult`：松手后最终文本，由 Hook 的 `onEnd` 内通过 `getFinalTranscript()` 得到并传入。

### 3.4 类型声明

- **文件**：`src/types/speech.d.ts`
- **作用**：为 `SpeechRecognition`、`SpeechRecognitionEvent`、`SpeechRecognitionErrorEvent` 等提供 TypeScript 类型（部分环境未在 lib.dom 中声明）。

## 4. 数据流与生命周期

### 4.1 一次「按住说话」的时序

1. 用户按下麦克风按钮 → `handlePointerDown` → `start()`。
2. Hook 内 `new SpeechRecognition()`，设置 `continuous=true`、`interimResults=true`、`lang`，注册 `onresult`/`onend`/`onerror`，调用 `rec.start()`。
3. 浏览器请求麦克风权限（若未授权）；用户同意后开始采集与识别。
4. 识别过程中：
   - 多次触发 `onresult`：`isFinal === false` 的通过 `onResult(transcript, false)` → 组件层 `onInterim(transcript)` → 父组件 `setVoiceInterim(transcript)`，界面显示「正在识别：xxx」并随说话更新。
   - 某段话结束会收到 `isFinal === true`，Hook 将该段追加到 `finalTranscriptRef`。
5. 用户松手（或指针移出）→ `handlePointerUp` / `handlePointerLeave` → `stop()` → `rec.stop()`。
6. 浏览器收尾当前语音，可能再触发一次 `onresult`（最后一段 final），然后触发 `onend`。
7. Hook 的 `onend` 中：`rec` 置空、`isListening` 置 false、调用传入的 `onEnd()`。  
   HoldToTalkButton 在 `onEnd` 里调用 `getFinalTranscript()` 得到整段文本，若有则 `onResult(text)`，再调用父组件的 `onEnd()`。
8. 父组件：`onResult` 将文本追加到输入框（如 `setText(prev => prev + text)`），`onEnd` 清空 `voiceInterim`，隐藏「正在识别」区域。

### 4.2 为何不在 `stop()` 里直接调用父级 `onEnd`

若在 `stop()` 中立刻清空状态并调用父级 `onEnd()`，此时浏览器可能尚未触发最后的 `onresult` 和 `onend`，`getFinalTranscript()` 可能不完整。因此约定：**只在该次识别的 `rec.onend` 里做清理并执行「松手后逻辑」**（取最终文本、回调父组件），保证拿到完整结果后再更新 UI。

## 5. 浏览器支持与限制

### 5.1 支持情况（概要）

- **Chrome / Edge**：支持较好，含 `continuous`、`interimResults`、多语言。
- **Safari**：支持（含 iOS 14.5+），部分行为可能略有差异。
- **Firefox**：对 `SpeechRecognition` 支持有限或默认关闭，实际可能不可用。

项目通过是否存在 `SpeechRecognition` / `webkitSpeechRecognition` 判断 `supported`，不支持时按钮置灰并提示「当前浏览器不支持语音输入」。

### 5.2 安全与权限

- **安全上下文**：麦克风仅在 **HTTPS** 或 **localhost** 下可用（部分浏览器对 file 协议限制更严）。
- **用户授权**：首次使用会弹出麦克风权限；用户拒绝则触发 `onerror`（如 `not-allowed`），可在 UI 上提示。

---

## 6. 错误处理

- **不支持**：无 `SpeechRecognition` 时，`supported === false`，按钮禁用并显示 `unsupportedTitle`。
- **未授权 / 拒绝**：`onerror` 收到 `not-allowed`，Hook 设置 `error` 并调用 `onError`；按钮可展示错误文案（如「未授权麦克风或已取消」）。
- **其他错误**：如 `no-speech`、网络或引擎错误，同样通过 `onerror` → `error` 状态与 `onError` 回调暴露，便于上层提示。

---

## 7. 扩展与配置

- **多语言**：通过 `HoldToTalkButton` 的 `lang` 传入，如 `zh-CN`、`en-US`，即可切换识别语言。
- **禁用**：父组件可传 `disabled`（如未连接时客服输入区禁用），按钮与识别均不触发。
- **样式**：麦克风按钮使用全局类名 `.hold-to-talk-btn`，录音中态为 `.listening`；「正在识别」使用 `.chat-session-voice-interim`，可在 `globals.css` 中统一调整。

## 8. useSpeechRecognition 源码逐段说明

下面按文件结构说明 `src/hooks/useSpeechRecognition.ts` 里每一块在做什么。

### 8.1 浏览器 API 检测（第 10–14 行）

```ts
const SpeechRecognitionCtor =
  typeof window !== 'undefined'
    ? (window as ...).SpeechRecognition ?? (window as ...).webkitSpeechRecognition
    : undefined;
```

- **目的**：拿到浏览器里的语音识别构造函数，用于后面 `new SpeechRecognitionCtor()`。
- **逻辑**：仅在存在 `window` 时（浏览器环境）查找；先试标准名 `SpeechRecognition`，没有再试 `webkitSpeechRecognition`（Safari 等）；SSR 时 `window` 不存在则为 `undefined`。
- **结果**：有则 Hook 可工作（`supported === true`），没有则 `supported === false`，调用方禁用按钮并提示不支持。

### 8.2 入参：Options（第 16–36 行）

- **`lang`**：识别语言，默认 `'zh-CN'`，会赋给 `rec.lang`。
- **`interimResults`**：是否要中间结果（边说边出字），默认 `true`，对应 `rec.interimResults`。
- **`continuous`**：是否在单次 start→stop 间持续识别多段，默认 `true`，对应 `rec.continuous`。
- **`onResult(transcript, isFinal)`**：每次有识别结果时回调；`isFinal === true` 表示该段已确认，`false` 表示中间结果。
- **`onEnd()`**：识别会话结束（松手后 `rec.onend` 或出错结束时）调用，用于清 UI、取最终文本等。
- **`onError(error)`**：出错时调用，如未授权、无语音等。

### 8.3 状态与 ref（第 38–41 行）

- **`isListening`**：当前是否在识别（已 `start()` 且尚未 `onend`），用于按钮高亮等。
- **`error`**：最近一次错误文案（如「当前浏览器不支持语音识别」「未授权麦克风或已取消」），用于 UI 提示。
- **`recognitionRef`**：当前 `SpeechRecognition` 实例。用 ref 是为了在 `stop()`、卸载清理里能拿到同一实例并调用 `stop()`/`abort()`，且不会因 state 更新导致重复创建。
- **`finalTranscriptRef`**：数组，在本次「按住期间」把每段 **最终结果**（`isFinal === true`）的文本 push 进去；松手后通过 `getFinalTranscript()` 把该数组 `join('')` 成整段文本返回，供输入框填入。

### 8.4 stop（第 44–53 行）

- **时机**：用户松手（或指针移出）时由 `HoldToTalkButton` 调用。
- **作用**：对当前识别实例调用 `rec.stop()`（不用 `abort()`，这样引擎会收尾并尽量再给一次最终结果，再触发 `onend`）。
- **为何不在这里清状态、不在这里调 onEnd**：清理和 `onEnd` 统一放在 `rec.onend` 里做，避免 `stop()` 返回时最后一段结果还没通过 `onresult` 传完，导致 `getFinalTranscript()` 不完整。

### 8.5 start（第 56–110 行）

1. **不支持则直接返回**：没有 `SpeechRecognitionCtor` 时设 `error` 并返回 `false`。
2. **清空上次结果**：`finalTranscriptRef.current = []`，`setError(null)`。
3. **创建并配置实例**：`new SpeechRecognitionCtor()`，设置 `continuous`、`interimResults`、`lang`。
4. **`rec.onresult`**：  
   - 遍历本次事件里从 `event.resultIndex` 到末尾的每一条结果。  
   - 每条有 `transcript` 和 `isFinal`。  
   - **最终结果**（`isFinal === true`）：拼到当次循环的 `lastFinal` 并 push 进 `finalTranscriptRef.current`，同时 `onResult(text, true)`。  
   - **中间结果**（`isFinal === false`）：只 `onResult(text, false)`，不写入 ref，用于界面「正在识别：xxx」实时展示。
5. **`rec.onend`**：识别会话结束（stop 触发的正常结束或异常结束都会进来）。这里把 `recognitionRef.current` 置空、`setIsListening(false)`、调用 `onEnd()`。上层在 `onEnd` 里会调 `getFinalTranscript()` 拿到整段文本并写入输入框。
6. **`rec.onerror`**：把 `event.error` 转成可读文案（如 `not-allowed` →「未授权麦克风或已取消」），设 `error` 状态、清 ref、`setIsListening(false)`、调用 `onError` 和 `onEnd`。
7. **`rec.start()`**：真正开始识别；成功后把实例存进 `recognitionRef`、`setIsListening(true)`、返回 `true`。若 `start()` 抛错则设 `error`、调 `onError`、返回 `false`。

### 8.6 卸载清理（第 113–124 行）

- **useEffect 清理函数**：组件卸载时若还有识别实例（`recognitionRef.current` 存在），则调用 `rec.abort()` 立即终止识别（不等待最终结果），然后置空 ref。
- **用 `abort()` 的原因**：卸载时不需要再把结果写进已不存在的输入框，直接中断即可，避免 `onend` 里再去操作已卸载的 state。

### 8.7 getFinalTranscript（第 127–129 行）

- **用途**：在本次「按住→松手」周期内，把所有已 push 进 `finalTranscriptRef.current` 的最终段文本按顺序拼成一段字符串并 trim。
- **调用时机**：由上层在 `onEnd` 回调里调用（例如 `HoldToTalkButton` 在收到 Hook 的 onEnd 后先 `getFinalTranscript()` 再把结果通过 `onResult` 传给输入框）。

### 8.8 返回值（第 131–138 行）

- **supported**：是否存在语音识别 API。
- **isListening**：是否正在识别。
- **error**：最近一次错误信息。
- **start / stop**：开始/结束识别。
- **getFinalTranscript**：取本次按住期间所有最终结果拼接成的完整文本。
