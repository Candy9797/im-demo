# 消息免打扰实现说明

本文说明聊天多人会话页（`/chat`）中「消息免打扰」功能的实现方式：状态、交互与展示，以及当前范围与扩展思路。

---

## 一、功能含义

**消息免打扰**：用户对某个群开启后，该群新消息不打扰用户（例如不响铃、不弹推送、或仅弱提示）。本项目为前端 Mock，只做「免打扰标记」与侧边栏展示区分，不涉及推送/声音/系统角标。

---

## 二、状态与动作（Store）

**位置**：`src/store/chatSessionStore.ts`

### 状态

| 字段 | 类型 | 含义 |
|------|------|------|
| `mutedGroupIds` | `string[]` | 已开启免打扰的群 ID 列表 |

初始值为 `[]`，刷新页面会重置（未持久化）。

### 动作

```ts
toggleMuteGroup: (groupId: string) => void
```

- **作用**：切换该群是否在免打扰列表中。
- **实现**：若 `groupId` 已在 `mutedGroupIds` 中则移除，否则追加。
- **写法**：通过 Immer 的 `set((state) => { ... })` 直接修改 `state.mutedGroupIds`（数组不可变更新）。

```ts
// 核心逻辑
toggleMuteGroup: (groupId) =>
  set((state) => {
    const idx = state.mutedGroupIds.indexOf(groupId);
    if (idx >= 0) state.mutedGroupIds = state.mutedGroupIds.filter((id) => id !== groupId);
    else state.mutedGroupIds = [...state.mutedGroupIds, groupId];
  }),
```

## 三、UI 入口与展示

### 1. 入口：群信息面板

**文件**：`src/components/chat/ChatGroupInfoPanel.tsx`

- 在「管理」区域提供 **「消息免打扰」** 开关（ checkbox ）。
- `checked` 由 `mutedGroupIds.includes(groupId)` 得到，`onChange` 调用 `toggleMuteGroup(groupId)`。
- 用户进入某个群 → 点击标题右侧「群信息」→ 在面板中勾选/取消勾选即可开启或关闭免打扰。

### 2. 展示：侧边栏群列表

**文件**：`src/components/chat/ChatSessionSidebar.tsx`

- 从 store 读取 `mutedGroupIds`，对每个群计算 `isMuted = mutedGroupIds.includes(g.id)`。
- **会话行**：若 `isMuted` 为 true，为该条会话增加 class `muted`，预览文案略弱化（见下样式）。
- **未读角标**：若该群免打扰且存在未读，角标使用 class `chat-session-unread muted`，显示为灰色样式，与未免打扰的红色角标区分。

**样式**：`src/app/globals.css`

```css
/* 免打扰会话：预览弱化 */
.chat-session-item.muted .chat-session-item-preview {
  color: var(--text-tertiary);
  opacity: 0.85;
}

/* 免打扰群的未读角标：灰色，弱提示 */
.chat-session-unread.muted {
  background: var(--bg-hover);
  color: var(--text-tertiary);
}
```

## 四、数据流小结

1. 用户点击群信息面板中的「消息免打扰」开关 → 调用 `toggleMuteGroup(groupId)`。
2. Store 更新 `mutedGroupIds`，Zustand 通知订阅方。
3. `ChatGroupInfoPanel` 的 checkbox 与 `ChatSessionSidebar` 的列表项/角标均依赖 `mutedGroupIds`，因此会同步更新：开关状态、列表行样式、未读角标颜色。

---

## 五、当前实现范围

| 项目 | 说明 |
|------|------|
| **状态** | 仅内存中的 `mutedGroupIds`，刷新即丢失。 |
| **表现** | 侧边栏免打扰群：预览略弱化、未读角标灰色；群信息面板内开关与状态一致。 |
| **未做** | 持久化（本地/服务端）、推送静音、声音开关、App 角标、红点策略等。 |

若要持久化，可把 `mutedGroupIds` 写入 localStorage 或在上线后同步服务端，在 store 初始化/登录后从本地或接口恢复。

---

## 六、扩展思路（与后端/推送对接）

在真实 IM 场景中，「免打扰」通常还会影响：

- **推送**：服务端根据用户维度的免打扰配置，决定是否发推送、是否静音。
- **声音**：前端或 Native 在收到新消息时，若该会话免打扰则不播放提示音。
- **角标**：系统/应用角标是否把该会话的未读数计入「总未读」或单独弱化展示。

当前项目仅在前端用 `mutedGroupIds` 做侧边栏与群信息面板的展示区分；后续可将该列表同步到服务端或本地配置，由推送/声音/角标逻辑按配置决定是否打扰用户。
