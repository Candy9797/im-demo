# Shop 文案 i18n：只维护一份，可选自动翻译

## 只维护中文，生成英文

1. **只改 `src/messages/zh.json`**  
   key 和占位符（如 `{count}`、`{id}`）保持不变。

2. **生成 en.json**  
   ```bash
   npm run i18n:translate
   ```
   - **未配置翻译 key**：会按 zh 的结构生成 `en.json`（内容先和 zh 一致），你本地把英文改好即可，或后续自己配置 key 再跑。
   - **自己配置 key 后**：设置 `LIBRE_TRANSLATE_API_KEY` 或 `DEEPL_AUTH_KEY` 再运行，会用对应接口自动翻译并写入 `en.json`（翻译服务需自申请/自付费，此处不提供 key）。

3. **可选环境变量**  
   `TRANSLATE_SOURCE`、`TRANSLATE_TARGETS`、`LIBRE_TRANSLATE_URL`、`LIBRE_TRANSLATE_API_KEY`、`DEEPL_AUTH_KEY` — 详见脚本顶部注释。

## 占位符

文案里的 `{count}`、`{id}` 等会在脚本里被保护，翻译时不会被改掉，生成的文件可直接给 `t('itemCount', { count })` 用。
